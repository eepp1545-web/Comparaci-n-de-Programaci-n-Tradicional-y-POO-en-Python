En la versión tradicional resolví el problema usando funciones: una recoge las temperaturas, otra calcula el promedio y otra muestra los resultados. Es un enfoque directo, práctico y suficiente para programas pequeños.
En la versión orientada a objetos organicé la información dentro de clases. Cada día se representa como un objeto y la semana completa se maneja desde otra clase. Esto vuelve el código más claro y permite agregar nuevas funcionalidades sin desordenarlo.
En pocas palabras:
La programación tradicional es rápida y simple para tareas puntuales.
La POO ofrece una estructura más ordenada y escalable cuando el proyecto puede crecer.
